# 调试流程

Bug 修复流程。

## 1. 复现

先定位失败命令、接口、页面或日志。

## 2. 收敛范围

只读取失败路径上的文件。

## 3. 假设

编辑前先判断最可能原因。

## 4. 修复

只做最小修复，不重构无关代码。

## 5. 验证

重新运行失败命令或相关测试。

Bug 修复必须优先新增或更新回归测试；如果当前项目缺少测试框架或无法稳定断言，必须在 AI 开发记录中说明原因、替代验证、剩余风险和后续补测建议。

## 6. 记录

将修复写入变更记录。

如果 Bug 修复涉及生产风险、安全、权限、数据、SQL、多个文件或多个模块，必须新增独立 AI 开发记录。

如果问题暴露出 Harness 规则缺口，必须按 `harness-engineering/governance/feedback-loop.md` 更新对应约束、技能或工作流。
