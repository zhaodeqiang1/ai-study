# 影响范围评估检查清单

需求设计时使用本清单，避免只写业务目标而漏掉实现影响。

## 模块影响

- 目标微服务模块。
- 是否涉及 `ruoyi-auth`、`ruoyi-gateway`、`ruoyi-common` 或根 `pom.xml`。
- 是否需要新增业务模块、远程 API 模块或前端目录。
- 是否影响已有系统管理能力。

## 接口影响

- 新增接口。
- 修改接口。
- 删除或废弃接口。
- 入参、出参、分页、排序、错误码。
- 兼容性和调用方影响。

## 前端影响

- 页面。
- API 封装。
- 路由。
- 按钮权限。
- 表单校验。
- 加载、空状态、错误状态。

## 数据影响

- 新表或字段。
- 索引。
- Mapper 查询。
- 数据迁移。
- 数据量和性能。
- 正向 SQL、回滚 SQL、验证 SQL。

## 权限和安全影响

- 菜单权限。
- 按钮权限。
- 角色授权。
- 数据权限。
- 输入校验。
- 敏感信息处理。
- 外部调用白名单。

## AI 功能影响

- 模型调用。
- Prompt 模板。
- RAG 检索。
- Embedding 和向量库。
- 工具调用。
- Token 统计。
- AI 审计日志。
- 失败降级。

## 测试和验证影响

- 单元测试。
- 回归测试。
- 集成验证。
- 前端构建。
- SQL 验证。
- AI 评测集。
- 无法验证的原因和风险。

## 发布和回滚影响

- 配置变更。
- 部署顺序。
- 灰度策略。
- 回滚脚本或步骤。
- 不可逆影响说明。
- 监控和日志。
