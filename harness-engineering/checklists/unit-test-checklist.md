# 单元测试检查清单

## 何时必须考虑单元测试

- 新增 Service 业务逻辑。
- 修改状态流转。
- 修改权限判断。
- 修改参数校验。
- 修改 Mapper 查询条件。
- 修复 Bug。
- 新增 Prompt 渲染、RAG 检索、模型参数组装、工具调用逻辑。

## 推荐测试范围

- 正常路径。
- 参数为空或非法。
- 权限不足。
- 数据不存在。
- 状态不允许。
- 外部依赖失败。
- 边界长度或数量限制。

## 后端测试建议

如果测试依赖已具备，优先使用：

- JUnit。
- Mockito。
- Spring Boot Test。

推荐命名：

```text
XxxServiceTest
XxxControllerTest
XxxMapperTest
```

推荐位置：

```text
<module>/src/test/java
```

## AI 功能测试建议

AI 相关逻辑至少覆盖：

- Prompt 变量缺失。
- 超长输入。
- 无检索结果。
- 模型调用失败。
- 工具调用不在白名单。
- 敏感内容输入。

## 无法补测试时必须记录

- 当前缺少测试框架或依赖。
- 测试需要真实数据库或外部服务。
- 当前变更只涉及文档。
- 当前变更无法稳定断言。

同时必须给出替代验证方式。
