# 安全约束

AI 相关功能默认按生产环境要求设计。

## 密钥安全

- 不允许把 API Key、Access Key、Secret 写入代码。
- 密钥必须来自配置中心、环境变量或安全配置。
- 示例配置只能使用占位符。

## 权限安全

管理接口必须考虑：

- 登录态。
- 角色权限。
- 菜单权限。
- 按钮权限。
- 部门或租户数据边界。

## 输入安全

所有用户输入都应视为不可信：

- 文本需要长度限制。
- 文件需要类型、大小、数量限制。
- URL 需要白名单或安全校验。
- SQL 相关能力必须使用白名单。

## 输出安全

AI 输出不能直接作为高风险操作依据。

涉及以下场景必须人工确认：

- 删除数据。
- 修改权限。
- 执行 SQL。
- 发送外部通知。
- 影响财务、法律、合同、隐私的内容。

## 日志安全

日志中不记录：

- 密钥。
- 完整令牌。
- 用户隐私信息。
- 原始敏感文档全文。
- 大模型完整 Prompt，除非已经脱敏。
