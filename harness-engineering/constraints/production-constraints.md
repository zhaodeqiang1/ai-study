# 生产约束

生产可用不是只让功能跑通，还需要可配置、可观察、可回滚。

## 配置

模型供应商、模型名称、接口地址、超时、重试、限流、向量库连接等必须可配置。

## 可观察性

AI 调用需要记录：

- 调用用户。
- 业务场景。
- 模型供应商。
- 模型名称。
- Prompt 版本。
- Token 使用量。
- 耗时。
- 成功或失败。
- 错误摘要。

## 容错

模型调用失败时应有明确处理：

- 返回可理解错误。
- 不影响主业务事务。
- 可重试任务应进入任务队列或标记失败状态。

## 回滚

涉及表结构、菜单、权限、配置的变更，必须提供回滚脚本或明确回滚步骤。

SQL/migration 必须包含：

- 正向 SQL。
- 回滚 SQL 或回滚方案。
- 数据影响。
- 部署顺序。
- 验证 SQL。
- 不可逆说明。

如果不可逆，必须说明原因、风险、备份方式和人工确认要求。

## 并发

长任务不要阻塞 HTTP 请求：

- 文档解析。
- 向量化。
- 批量导入。
- 批量评估。

这些任务应考虑异步、队列、定时任务或状态机。
