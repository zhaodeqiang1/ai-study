# Token 预算

本文件用于限制 AI 上下文读取，减少 Token 消耗。

## 默认读取预算

一次普通任务最多读取：

- Harness 文件：5 个以内。
- 业务源码文件：12 个以内。
- 配置文件：5 个以内。
- SQL 文件：5 个以内。
- 测试文件：6 个以内。

超过预算前，AI 必须说明为什么需要继续读取。

## 默认输出预算

最终回复应保持简短，只包含：

- 变更摘要。
- 关键文件路径。
- 验证结果。
- 已知风险。

## 降低 Token 的方法

- 先读 `AGENTS.md`，再读渐进加载文件。
- 只读取目标模块。
- 只读取同类实现作为参考。
- 不粘贴大段源码。
- 对长文件只定位相关方法或片段。
- 不重复解释通用框架知识。

## 允许扩大上下文的情况

- 编译失败需要追踪跨模块依赖。
- 接口契约涉及 `ruoyi-api`。
- 权限、菜单、路由需要前后端联动。
- 数据库字段影响 Mapper、实体、页面。
- 生产风险无法在当前上下文判断。
