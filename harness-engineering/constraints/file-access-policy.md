# 文件访问策略

本文件定义 AI 读取和修改文件的边界。

## 读取策略

优先读取：

1. 用户明确指定的文件。
2. 当前任务目标模块。
3. 同模块同类文件。
4. 与编译或接口契约有关的配置。

谨慎读取：

- 根目录 `pom.xml`。
- `ruoyi-common`。
- `ruoyi-auth`。
- `ruoyi-gateway`。
- 大型 SQL 初始化文件。

## 修改策略

默认允许修改：

- 新增 Harness 工程约束文件。
- 新增 Harness 独立业务模块。
- 用户明确指定的业务文件。

默认禁止修改：

- 认证流程。
- 网关路由。
- 全局安全过滤器。
- 公共工具类。
- 已有数据库初始化主脚本。
- 与任务无关的格式化内容。

## 扩权条件

只有满足以下条件之一，AI 才能扩大修改范围：

- 用户明确要求。
- 当前模块无法独立完成任务。
- 编译错误证明必须调整上游契约。
- 安全问题必须在公共入口修复。

扩大范围前，AI 应先说明原因。
