# Spring AI Alibaba 技能

## 适用场景

当任务涉及大模型调用、Embedding、RAG、向量检索、工具调用、Prompt 模板、模型配置时读取本文件。

## 设计原则

- 模型调用能力要封装在独立服务中。
- Prompt 模板应可版本化、可审计。
- 模型参数应可配置，不硬编码。
- RAG 回答应尽量返回引用来源。
- 用户输入必须有长度和安全限制。
- 模型输出不能直接驱动高风险操作。

## 推荐分层

- `ai/chat`：聊天和流式输出。
- `ai/prompt`：提示词模板和变量渲染。
- `ai/rag`：检索增强生成。
- `ai/vector`：Embedding 和向量库适配。
- `ai/tool`：工具调用定义。
- `log`：模型调用日志和 Token 统计。

## RAG 流程

推荐闭环：

1. 上传文档。
2. 解析文本。
3. 文档切片。
4. 生成向量。
5. 写入向量库。
6. 用户提问。
7. 向量召回。
8. 拼接上下文。
9. 调用模型。
10. 返回回答和引用来源。
11. 记录日志和反馈。

## 生产要求

- 模型供应商、模型名、Key、端点、超时必须可配置。
- Token 使用量、耗时、失败原因必须可记录。
- 文档处理和向量化应异步化。
- 对外部工具调用必须有白名单和权限控制。
